=head1 NAME

Compatibility - Where things went right

=head1 Description

When translating FLTK's I<very> C++ API to Perl, I've been faced with a
few decisions. Here's a list of many of them:

=over

=item C<FLTK::parsecolor ( COLOR, [ LENGTH ] )>

The C++ function takes two parameters:
C<parsecolor( char * name, unsigned length )> ...but this is Perl and
forcing you to call C<parsecolor( $mycolor, length($mycolor) )> isn't.

=back
